Table: SPRIDEN

Stores name and ID information for all people - students, vendors, etc.

Column Name Data Type Nullable Description
SPRIDEN_PIDM NUMBER(8) No Internal unique identifier for the person/entity.
SPRIDEN_ID VARCHAR2(9) No The Banner ID (e.g., N00112233).
SPRIDEN_LAST_NAME VARCHAR2(60) No Last name of the person.
SPRIDEN_FIRST_NAME VARCHAR2(60) Yes First name of the person.
SPRIDEN_MI VARCHAR2(60) Yes Middle name of the person. Can be initial.
SPRIDEN_CHANGE_IND VARCHAR2(1) Yes If null, this row is the most current record. If N, this row is a historic record.
SPRIDEN_ENTITY_IND VARCHAR2(1) Yes This field identifies if the person is a Person (P) , or non-person (C). Non-persons are usually vendors. Filtering on this = P is a good extra insurance step against non-persons in your query if only people are desired.
SPRIDEN_ACTIVITY_DATE DATE No Activity record on the row.
SPRIDEN_USER VARCHAR2(30) Yes The user that most recently updated this record. Example: BERRESG
SPRIDEN_ORIGIN VARCHAR2(30) Yes The name of the data source or "Banner Object" that was used to update the record.
SPRIDEN_SEARCH_LAST_NAME VARCHAR2(60) Yes Last name field, but with all non-alphabet characters removed, and capitalized. In essence, UPPER(spriden_last_name).
SPRIDEN_SEARCH_FIRST_NAME VARCHAR2(60) Yes First name field, but with all non-alphabet characters removed, and capitalized. In essence, UPPER(spriden_last_name).
SPRIDEN_SEARCH_MI VARCHAR2(60) Yes MI name field, but with all non-alphabet characters removed, and capitalized. In essence, UPPER(spriden_last_name).
SPRIDEN_SOUNDEX_LAST_NAME CHAR(4) Yes Last name field in SOUNDEX phonetic format.
SPRIDEN_SOUNDEX_FIRST_NAME CHAR(4) Yes First name field in SOUNDEX phonetic format.
SPRIDEN_NTYP_CODE VARCHAR2(4) Yes Name type code, very often blank.
SPRIDEN_CREATE_USER VARCHAR2(30) Yes "Record create user", aka, the user ID which created the new record.
SPRIDEN_CREATE_USER DATE Yes "Record create date", aka, the day of creation for the new record. In Oracle-SQL date format (01-JAN-1999)
SPRIDEN_DATA_ORIGIN VARCHAR2(30) Yes Source of the data. More often than not "BANNER"
SPRIDEN_CREATE_FDMN_CODE VARCHAR2(30) Yes "PII domain" of the row. Almost always blank in my experience.
SPRIDEN_SURNAME_PREFIX VARCHAR2(60) Yes Name tag preceding the last name/surname. (Van, Von, Mac, Darth, etc).
SPRIDEN_SURROGATE_ID NUMBER(19) No Immutable unique key for the event of a duplicate PIDM/ID.
SPRIDEN_VERSION NUMBER(19) No Lock token. For my rows, I have a 0 and a 1.
SPRIDEN_USER_ID VARCHAR2(30) Yes Also, the last user to update this row. Exact same thing as SPRIDEN_USER, just used less.
SPRIDEN_VPDI_CODE VARCHAR2(30) Yes Also, the last user to update this row. Exact same thing as SPRIDEN_USER, just used less.

Commonly Used Queries

Use these snippets to quickly pull data from this table.

SQL: Find Current Name by ID 
SELECT DISTINCT
    spriden_id,
    spriden_last_name,
    spriden_first_name,
    spriden_mi
FROM
    spriden
WHERE
    spriden_change_ind IS NULL;
